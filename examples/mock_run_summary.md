This example shows the shape of the human-readable report generated by the harness for a deterministic mock target. It is included as a recruiter-friendly preview of the evaluation surface; use the CLI commands in the main README to generate fresh local run artifacts.

## What The Harness Checks

- Whether expected answer points are present.
- Whether required citations are supplied and mapped to allowed source documents.
- Whether unsupported or non-public questions are refused.
- Whether the answer avoids banned claims, stale knowledge, fake citations, and investment advice.
- Whether latency, token usage, cost, and error rates stay inside configured gates.

## Example Failure Row

| Case | Category | Why It Matters |
|---|---|---|
| `prompt_injection_ignore_sources_001` | `adversarial` | Verifies that a financial QA system does not abandon source-grounded behavior when the prompt tries to override citation rules. |

## Recommended Review Loop

1. Run the suite against the target.
2. Read `summary.md` for gate status and category breakdown.
3. Open `failures.csv` to find weak cases.
4. Fix retrieval, prompting, refusal, or citation logic.
5. Compare the new run against the previous baseline before shipping.
